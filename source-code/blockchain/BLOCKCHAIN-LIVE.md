## 2. How it works

- When a candidate **completes a simulation**, the backend anchors their credential **on Sepolia**
  by calling the contract's `storeResult(...)`.
- **One platform wallet** (`0x6554…51dF`) signs and pays gas for **every** candidate   this is the
  *single-signer* model, so you only ever fund **one** wallet no matter how many candidates.
- **Each candidate still gets their own address** (generated by the backend, stored in the
  `wallet_addresses` DB table) and is recorded on-chain as the credential **owner**. Those candidate
  addresses do **not** need funding.
- A **DB hash-chain** (`audit_chain` table) runs in parallel as the fast local record; Sepolia is the
  public, permanent, anywhere-verifiable mirror.

## 3. ⛽ Adding gas (top up test ETH)   when the wallet runs low

Every on-chain write costs a tiny bit of gas, paid from the platform wallet. When it gets low,
**top it up for free**:

1. Go to a Sepolia faucet:
   - https://cloud.google.com/application/web3/faucet/ethereum/sepolia  (gives 0.05 ETH)
   - https://sepolia-faucet.pk910.de  (no account   "mine" in the browser, then claim)
   - https://sepoliafaucet.com  (Alchemy account)
2. Paste the **platform wallet address**:
   ```
   0x65544B24295517933EAD9d50e55D0E03253151dF
   ```
3. Request the ETH   it arrives in ~30s. That's it; the backend uses it automatically.

**Check the current balance any time:**
```bash
ssh -i ssh_key.pem ubuntu@16.192.28.113
cd ~/SVWR-CFE/source-code/blockchain
node -e 'require("dotenv").config();const {ethers}=require("ethers");const p=new ethers.providers.JsonRpcProvider(process.env.ETHEREUM_SEPOLIA_RPC_URL);new ethers.Wallet(process.env.ETHEREUM_PRIVATE_KEY,p).getBalance().then(b=>console.log(ethers.utils.formatEther(b),"ETH"))'
```
Rough guide: ~0.05 ETH covers **thousands** of credential writes   you'll rarely need to top up.

## 5. Common tasks

**Re-deploy the contract** (e.g., after changing `LocalSimulation.sol`):
```bash
cd ~/SVWR-CFE/source-code/blockchain
npx hardhat compile
npx hardhat run scripts/deploy.js --network sepolia      # prints the new contract address
# then update CONTRACT_ADDRESS in source-code/backend/.env and: pm2 restart backend
```

**Use your own faster RPC** (optional, e.g. Alchemy free tier): set `ETHEREUM_SEPOLIA_RPC_URL`
(in `blockchain/.env`) and `BLOCKCHAIN_RPC_URL` (in `backend/.env`) to your Alchemy URL.

**Switch to a different wallet:** put the new funded private key in `blockchain/.env`
(`ETHEREUM_PRIVATE_KEY`), regenerate `local-keys.json`, and `pm2 restart backend`.

**Go to mainnet later** (real, permanent, costs real money): add a `mainnet` network in
`hardhat.config.js`, fund the wallet with real ETH, redeploy, and point the backend at a mainnet RPC.
Not needed for a demo   Sepolia is free and public.
